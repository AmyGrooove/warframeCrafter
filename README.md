# Warframe Prime Tracker

Личное SPA-приложение на `Vite + React + TypeScript` для учёта прайм-предметов Warframe. Приложение хранит локальный инвентарь в браузере, подтягивает цены с `warframe.market`, считает выгоду обмена на дукаты и помогает отслеживать прогресс по mastery-предметам.

## Что умеет приложение

- искать и добавлять в инвентарь только прайм-предметы из каталога `warframe.market`
- хранить количество предметов, выбранный язык и прогресс освоения в `localStorage`
- показывать стоимость инвентаря по `min sell`, `max buy` и общей сумме по каждой позиции
- автоматически подгружать и кэшировать цены во вкладках стоимости и дукатов
- считать выгоду обмена на дукаты по метрике `дукаты / мин. продажа`
- вести отдельный каталог mastery-предметов с фильтрами по категории, Prime-статусу и освоению
- импортировать инвентарь и прогресс освоения из JSON
- полностью очищать локальные данные приложения одной кнопкой

## Разделы интерфейса

### Инвентарь

- поиск по английским и русским названиям
- карточки для быстрого добавления предметов
- ручное изменение количества
- удаление позиции из инвентаря
- фильтр `Все / Освоено / Не освоено`

### Стоимость

- автообновление цен для отображаемых предметов
- ручное обновление одной позиции или всего списка
- сортировка по количеству, `min sell`, `max buy`, сумме и времени обновления
- быстрые действия: отметить 1 штуку проданной, открыть страницу предмета на `warframe.market`

### Дукаты

- таблица предметов с ненулевой ducat-стоимостью
- сортировка по количеству, дукатам, общей сумме дукатов, `min sell`, `дукаты / платину`, времени обновления
- тот же фильтр по статусу освоения, что и в таблице стоимости

### Освоенные предметы

- объединённый mastery-каталог по категориям: варфреймы, компаньоны, оружие, archwing и другие группы
- фильтры по статусу `Не освоено / Освоено / Любой статус`
- фильтры `Prime / Не Prime / Любые`
- локальное переключение статуса освоения
- ссылки на вики для предметов, где они доступны
- постепенная догрузка карточек большими чанками

### Настройки

- переключение языка названий `RU / EN`
- импорт инвентаря из JSON
- импорт mastery-прогресса из JSON
- полная очистка локального состояния и кэшей

## Источники данных

### Каталог прайм-предметов и цены

- каталог берётся из `warframe.market`
- в каталог попадают только предметы с `Prime` в английском названии
- из каталога исключаются `Relic` и `Prime Access`
- цены берутся из top-order endpoint'а и считаются по онлайн-продавцам/покупателям, если они есть

### Mastery-каталог

- базовый каталог собирается из `warframe-items` JSON через `jsDelivr`
- живые данные и русские переводы добираются из `api.warframestat.us`
- компоненты, blueprint-части, реликвии, косметика и другие не-mastery сущности отфильтровываются на уровне нормализации

## Кэш и локальное хранение

- инвентарь, язык интерфейса и mastery-прогресс сохраняются в `localStorage`
- каталог прайм-предметов кэшируется на `24 часа`
- mastery-каталог кэшируется на `24 часа`
- price snapshot кэшируется на `30 минут`
- запросы цен идут через очередь с ограничением параллелизма, а после ошибок API включается временный cooldown на повторный запрос

## Форматы импорта

Импорт инвентаря:

```json
[
  { "name": "Revenant Prime Systems Blueprint", "count": 2 },
  { "name": "Fang Prime Blade", "count": 1 }
]
```

Импорт mastery-прогресса:

```json
[
  { "name": "Excalibur", "isMastery": true },
  { "name": "Carrier Prime", "isMastery": false }
]
```

Особенности импорта:

- сопоставление имён нечувствительно к регистру и части пунктуации
- можно импортировать как английские, так и русские названия, если они есть в каталоге
- дубликаты в файле схлопываются по предмету
- некорректные записи пропускаются, а итог показывается в интерфейсе

- `React 19`

- `Vite 8`

## Proxy и переменные окружения

По умолчанию фронтенд использует:

```txt
/api/warframe-market/v2
```

В `vite.config.ts` настроен dev-proxy:

- `/api/warframe-market/*` -> `https://api.warframe.market/*`

Это значит:

- в режиме `pnpm dev` дополнительная настройка не нужна
- если публиковать только статический `dist/`, нужен такой же proxy на хостинге или свой backend endpoint

Можно переопределить базовый URL через переменную окружения:

```env
VITE_WARFRAME_API_BASE=https://your-host.example.com/api/warframe-market/v2
```

## Ограничения

- приложение полностью клиентское: без аккаунтов, синхронизации и общей базы
- без proxy production-сборка не сможет безопасно ходить в `warframe.market` так же, как dev-режим
- mastery-переводы зависят от внешних источников, поэтому часть названий может оставаться на английском
- `site.webmanifest` добавлен, но сервис-воркера и offline-режима в проекте нет
